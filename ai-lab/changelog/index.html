Release timeline

Newest first.

  1. 2026-07-11 Build 05

    Corpus-Backed Secure RAG Retrieval

    The Secure RAG demo now retrieves from the generated structured public corpus — the same artifact rendered by the Corpus Explorer. The hand-authored content index is gone; the engine keeps only claim-status and search-alias metadata on top of the generated chunks.

    • One public-content artifact shared by the explorer and retrieval engine
    • Structural page and anchor validation for every generated result link
    • build_corpus.py --check fails QA when page content and the committed artifact drift
    • Eval expectations re-captured from real corpus-backed output; 42/42 checks passing locally
    • Still local keyword ranking — no embeddings, semantic benchmark, or production RAG claim

    Commit: 8775e00

  2. 2026-07-10 Build 04

    Structured Public Corpus + Explorer

    Step 1 of the Secure RAG production roadmap, shipped: an allowlist-first, stdlib-only ingestion script converts 8 approved public pages into section-level JSON chunks with stable IDs, provenance anchors, and content hashes. The explorer page renders every chunk and runs boundary checks live against the artifact.

    • private/ excluded by construction — the allowlist is the only input the script reads
    • Duplicate chunk IDs fail the build instead of shipping silently
    • Eval suite grew to cover the new corpus entry; all checks passing at ship time

    Commits: 3c3cf47

  3. 2026-07-10 Build 03

    Retrieval Eval Harness

    A regression eval suite that runs live in the browser against the same engine the Secure RAG demo uses — no mocks. Guardrail refusals, false-positive checks, hit@k retrieval, honesty states, and contract invariants, every result measured at page load.

    • Every expectation captured from real engine output before being locked in
    • Over-blocking fails the suite too — security questions must stay answered
    • Framed honestly as regression testing, not a semantic benchmark

    Commits: a9a7f4c

  4. 2026-07-10 Hardening

    Secure RAG deepening + accessibility pass

    The Secure RAG demo gained a visible threat model, a six-layer architecture map, and a prototype-vs-production honesty panel. A parallel pass fixed pressed-state semantics and labels across the demo calculators and the start-project chooser.

    Commits: b953a25, 61a4279, b373919

  5. 2026-07-05 Case study

    Secure RAG case study published

    The first full engineering write-up: problem, system design, security and claim-safety decisions, QA evidence including what review caught, lessons, and resume bullets — documented like a release, including the QA environment's limitations.

    Commits: ff4f1dc, 39b1459, d178c75

  6. 2026-07-03 → 07-05 Build 02

    Secure RAG Portfolio Search

    A security-aware retrieval prototype over public portfolio content: guarded queries, ranked results with claim-safety labels, visible pipeline states, and honest refusal/empty/offline states — with embeddings and the vector index explicitly labeled planned, not done.

    • Extraction-intent guardrail and hard blocks for private-data queries
    • Route whitelist so results can only point at known internal pages
    • Claim-accuracy and guardrail-hardening polish two days after first ship

    Commits: e2b433b, 432d848

  7. 2026-06-30 → 07-01 Lab launch

    Neo Labs page ships

    The AI Lab becomes a first-class page: the live assistant documented as a case study, the roadmap split into shipped / learning / planned, and the honesty rule set in writing — every claim labeled Live, Prototype, Planned, or Learning.

    Commits: 2b20162, ff6320e

  8. 2026-06-27 → 06-29 Build 01

    AI Portfolio Assistant goes live

    The first proof: a real assistant on the public site. Browser UI calls a same-origin /api/chat serverless function on Vercel; the provider key lives in server-side environment variables only, and a rule-based fallback keeps the feature honest when no backend is available.

    • Front-end assistant first, then the AI-ready serverless architecture, then live activation
    • Backend state detection and error surfaces fixed within two days of going live
    • Route whitelist, prompt constraints, and a reply sanitizer from day one

    Commits: c185526, d5bb06f, d01b95e, 31bccc2, 104953f
