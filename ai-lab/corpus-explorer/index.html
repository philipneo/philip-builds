Structured content, step 1 of the RAG roadmap

Structured Public Corpus.

The fourth Neo Labs build: the site's public pages converted into structured JSON chunks with stable IDs and content hashes by a build-time ingestion script. This is the first step the Secure RAG case study names on the path to real embeddings — shipped, explorable, and honest about what it is.

Honest scope: this is a manual, build-time script over an explicit allowlist of public pages — not a live ingestion pipeline. Chunks are section-level, not embedding-optimized. Nothing under private/ can enter the corpus, because the allowlist is the only input the script reads.

    How it's built

    Allowlist in, structured chunks out.

    scripts/build_corpus.py is a dependency-free Python script, run manually at build time. The allowlist-first design is the point: corpus approval happens in code review, not in a filter that could fail open.

    1 · Approve Explicit page allowlist Only listed public pages are readable. private/ is excluded by construction.
    →
    2 · Parse Sectionize each page Sections with stable anchors become chunks; heading + body text extracted, scripts and styles skipped.
    →
    3 · Identify Stable IDs + hashes IDs derive from page path + anchor, so they survive copy edits. A content hash tracks what changed.
    →
    4 · Emit JSON + JS artifacts corpus.json for humans and tooling; corpus.js so this page loads without a backend.
    Shipped in this step
    • Allowlist-first ingestion over public pages only
    • Section-level chunks with stable IDs and provenance anchors
    • Content hashes for change detection between builds
    • Boundary checks computed live on this page
    • Duplicate-ID guard that fails the build, not silently
    • The Secure RAG demo retrieves from this corpus — one artifact, two consumers
    Still ahead — labeled honestly
    • Embedding privacy review before any chunk is embedded
    • Chunk-size tuning for embedding models (sections are a first cut)
    • Real embeddings and a vector index with access control
    • Automated rebuilds — today the script runs manually
