Secure RAG Portfolio Search.

A working front-end prototype of a retrieval layer over this site's public portfolio knowledge: guarded queries, ranked sources, claim-safety labels on every result, and a visible security model — the next Neo Labs step toward real embeddings and a vector index.

Honest scope: this is a simulated retrieval layer that runs entirely in your browser. Ranking is local keyword scoring over a hand-curated index of public site content — no embeddings, no vector database, and no model calls yet. Those layers are on the planned architecture below and are labeled as planned, not done.

Security & trust layer

Guardrails you can see, not just claim.

The interesting part of RAG for real businesses is not the search — it is what the system refuses to do. This prototype makes those refusals visible.

Enforced in this prototype
  • No secrets in the corpus — the index is built only from public page content, so there is nothing sensitive to leak.
  • No private files indexed — internal operating notes under private/ are excluded by construction, not by filter.
  • Extraction-intent guardrail — queries that ask to reveal keys, secrets, credentials, or private data are refused with a clear reason.
  • Route whitelist — result links can only point to known internal pages of this site.
  • Safe error states — offline, too-broad, and no-result states never invent an answer.
  • Session-only logging — the audit preview lives in memory and clears on reload; nothing is sent anywhere.
Planned controls — not built yet
  • Access control on a real retrieval API, so private corpora could ever be considered safely.
  • Persistent audit logs with a review workflow instead of an in-memory preview.
  • Embedding privacy review before any real content is embedded into a vector index.
  • Prompt regression tests for the future answer-synthesis layer.
  • Retrieval eval set so relevance is measured, not eyeballed.

What this proves

A rehearsal for the real system — labeled as one.

  • I can design a retrieval product UX with honest states: ranked results, refusals, empty states, and fallbacks.
  • I can put security thinking in front of the feature — guardrails, whitelists, and a no-secrets corpus — before any real data is involved.
  • I can separate what is live, what is simulated, and what is planned without blending them.
  • I can stage a real RAG build (embeddings → vector index → synthesis → evals) as labeled future work instead of pretending it exists.
