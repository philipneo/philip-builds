Real AI readiness

Model-powered, without leaking keys.

The assistant is built to run two ways: rule-based fallback everywhere, and model replies only when the same-origin Vercel backend confirms it is connected.

Fallback

Rule-based guide

Routes visitors to demos, the matcher, and Start Project. No key required; works on GitHub Pages.

Connected

Vercel AI backend

If /api/chat responds successfully, the chat can render real model replies through the server.

Guarded

Serverless function

api/chat.js reads the key from an env variable only. Browser code never receives a key.

AI safety checklist

  • No secret in any browser-served file — keys live in env only
  • Frontend sends no personal data (message + page + last-match label)
  • System prompt forbids invented clients/results and promises
  • No automatic emails or bookings
  • Backend returns 503 (disabled) until a key is configured
  • API failure never breaks the site — it falls back to the guide

Enable it on Vercel

Deploy the frontend + api/chat.js on Vercel, set OPENAI_API_KEY in the Vercel dashboard, and keep shared/ai-config.js pointed at /api/chat. Full steps live in docs/AI_SETUP.md.

AI readiness test

Try the assistant — here's what should happen.

Open the assistant (bottom-right) and try these. Today every answer is the guided, rule-based response; model answers only appear after a backend is deployed and connected.

Test prompts

  • "What can you build?" → a plain-English list of build types
  • "Which demo should I open first?" → a recommendation + matcher link
  • "I run a cleaning business" → routes to the Cleaning Quote Calculator
  • "Match me" → opens the Project Matcher
  • "Are you real AI?" → an honest answer about the current mode

Modes & safety

  • Fallback mode: Rule-based fallback (badge in the chat header)
  • Connected mode: needs Vercel deploy + env key + working /api/chat
  • GitHub Pages is static — it cannot execute api/chat.js
  • No secret is ever in the browser; /api/chat runs only on the server host
  • No personal data is collected and no emails are sent automatically

AI control center

Memory, limits, and the activation path.

Remembers (this device only)

  • Your last matched demo + build
  • Nothing else — and you can clear it anytime

Never stores

  • Names, emails, or phone numbers
  • Cookies, tracking IDs, or chat history on a server
  • Any API key (those live only in deployment env)

Can't do on GitHub Pages

  • Run the serverless function
  • Call a model or hold a key
  • Edit or deploy itself

Next AI activation checklist

  • Set OPENAI_API_KEY in Vercel environment variables
  • Keep shared/ai-config.js set to endpoint: "/api/chat", allowModelMode: true
  • Test the assistant; confirm graceful fallback if the backend errors
