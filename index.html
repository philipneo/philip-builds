How I work

Shipped like releases, not homework.

Every Neo Labs project goes through the same pipeline: build it, deploy it, test it (accessibility, mobile, secret scans, functional tests), document it as a case study, and label every component Live, Prototype, Planned, or Learning. If a QA pass catches a defect, the case study says so — because "16 of 16 tests, one overflow caught and fixed" is worth more than any adjective.

01 Build Static-first, no libraries, security decisions before features.
→
02 Ship Deployed to the live site — demos you can open, not screenshots.
→
03 Test Functional tests, accessibility checks, mobile passes, secret scans.
→
04 Document A case study with architecture, QA evidence, and lessons.
→
05 Label Live / Prototype / Planned / Learning — nothing overclaimed.
